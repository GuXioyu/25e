## 2. 题目背景和当前范围

项目对应 2025 年全国大学生电子设计竞赛 E 题“简易自行瞄准装置”。系统由自动寻迹小车、二维云台、相机、405 nm 蓝紫激光、RDK X5 和 MSPM0（下文简称 M0）组成。

目标靶是竖直放置的 A4 紫外感光纸，四周贴黑色胶带，中心和同心圆由红色记号笔绘制。当前视觉使用黑胶带内部的白色矩形边界，实测尺寸统一为：

```text
27.0 cm × 18.0 cm
```

与当前视觉实现直接相关的目标：

- 基本要求（2）：2 s 内完成瞄准，再发射激光击中靶心；
- 基本要求（3）：指定位置、任意初始瞄准方向，4 s 内完成瞄准；
- 发挥部分（1）（2）：小车运动一圈或两圈时，激光连续照射并跟踪靶心；
- 静态题光斑痕迹距靶心最大偏差不超过 2 cm；
- 发挥一圈不超过 20 s，两圈不超过 40 s。

当前不实现：

- 发挥部分（3）的半径 6 cm 同步画圆；
- M0 内部 PID、四电机调度、循迹和串口屏具体代码；
- UART 校验、有效位、帧序号、时间戳或二进制协议改造。

紫外感光纸上的旧光痕会保留约 20～60 s。因此基础题必须“先瞄准，再开激光”，不能通过边照射边搜索完成粗瞄准。

## 3. 已确认的机械与硬件条件

### 3.1 相机和激光

- 相机和激光安装在同一个二维云台上，会同步转动；
- 相机位于激光上方；
- 相机光心与激光出光点的竖直安装距离记为 `set_lenth_cm`；当前记录为 `3.0 cm`；
- 水平方向机械安装偏差近似为 0；
- 相机光轴与激光轴近似平行，但不假定绝对平行；
- 激光上电时机械方向为水平；
- 激光出光点与靶心处于同一物理高度；
- 实测激光在 640×360 图像中的横坐标基本固定为 `x=325 px`；
- 激光纵坐标会随靶面距离变化。

`set_lenth_cm=3.0` 当前只作为机械安装事实保存在 `Stp23lConfig`。运行时 `predict_y` 使用实测多项式，该多项式已经吸收安装高度、微小轴线误差和相机投影的综合影响，不能再额外叠加一次 `fy×set_lenth/dis`，否则会重复补偿。

### 3.2 RDK、M0和电机

- RDK X5 负责全部视觉、PnP、STP23L读取、距离融合、预测坐标和UART输出；
- M0 负责题号、串口屏、小车循迹、四电机控制、云台动作、激光开关和控制闭环；
- M0 使用串口和电机地址控制四台电机；
- 云台电机为 ZDT X42S 第二代闭环步进电机；
- X42S 内部已有位置闭环，M0只需要实现视觉外环和目标位置更新；
- RDK不直接控制电机，M0不处理图像、PnP或激光点识别。

### 3.3 当前板端设备和运行参数

- 相机输入：`640×360`，请求 `60 FPS`；
- 主UART：`/dev/ttyS1`，`115200`；
- STP23L串口：`/dev/ttyS7`，`230400`；
- STP23L读取间隔：`0.02 s`；
- 网页显示模式当前为 `web`，WebSocket端口 `8082`；
- YOLO模型：`best_bayese_640x640_nv12.bin`；
- 当前靶框实时算法配置为 `canny`。

## 4. 坐标、单位和关键名词

- 图像坐标：x向右为正，y向下为正；
- 对外坐标均为 640×360 原图像素；
- 所有物理长度和距离统一使用 cm；
- UART距离保留一位小数；
- `center_x/y`：靶心像素；
- `nowlaser_x/y`：当前或最后有效激光点像素；
- `predict_x/y`：根据当前融合距离预测的激光轴像素；
- `laser_dis`：STP23L原始稳定距离，只在RDK内部快照和终端显示；
- `dis`：PnP/STP23L融合后当前采用的距离，对外发送给M0；
- `snapshot`：视觉线程与UART线程之间的一致数据快照，不是图像。

## 5. 当前总体数据流

```text
USB相机
  -> 未锁定时YOLO搜索靶子
  -> 锁定后ROI内Canny检测27×18cm白色内框
  -> 对角线交点得到center
  -> 靶面透视区域内检测nowlaser
  -> 每3视觉帧或强制事件执行PnP
  -> 独立线程读取并滤波STP23L
  -> PnP/STP23L融合得到dis
  -> predict_x固定325，predict_y使用三次拟合
  -> publish_snapshot保存同一时刻的最新数据
  -> UART每视觉帧发送最新快照
  -> M0根据题目模式使用相应字段控制电机和激光
```

`main.py` 中 `user_after_inference()` 已简化为三步：

1. `read_laser_distance()`读取STP23L距离；
2. 定位当前内框；
3. 分流到 `handle_target_missing()` 或 `handle_target_found()`。

有效目标、丢标处理和终端打印已经分成独立函数，避免把所有逻辑堆在回调中。

## 6. 靶框和靶心

### 6.1 靶框

当前使用白色内框四角作为PnP几何点，必须确保视觉检测到的边界与物理 `27.0×18.0 cm` 是同一条边界。若误用黑胶带外框或尺寸测量不一致，PnP距离会产生固定比例误差。

当前内框检测流程：

- YOLO只负责发现粗略靶框；
- 锁定后在ROI内执行Canny和四边形筛选；
- ROI外扩 `60 px`；
- 连续 `2` 帧找不到内框后解除跟踪，重新运行完整YOLO；
- 候选宽高比允许范围 `1.05～2.30`；
- 最低矩形度 `0.72`。

### 6.2 靶心

靶心不能使用四角平均值。当前 `calc_diagonal_center()`：

1. 将四角排序为左上、右上、右下、左下；
2. 计算两条对角线交点；
3. 将交点作为真实投影靶心。

原因：透视投影保持直线交点，但四角算术平均值在梯形透视下不等于物理矩形中心投影。

## 7. PnP视觉距离

`distance.py` 使用 OpenCV `solvePnP(..., SOLVEPNP_IPPE)`，物理点为：

```text
(-13.5, -9.0, 0) cm
( 13.5, -9.0, 0) cm
( 13.5,  9.0, 0) cm
(-13.5,  9.0, 0) cm
```

相机内参来自1280×720标定，并按运行分辨率缩放：

```text
fx = 1242.14572
fy = 1199.17793
cx = 654.440159
cy = 371.131499
```

畸变参数：

```text
(-0.378362253, -0.309338417, -0.00110892155, 0.000191972084, 1.71971762)
```

PnP默认每 `3` 个视觉帧更新一次。以下情况立即更新，不等待周期：

- 第一次锁定；
- 丢标后重新捕获；
- 靶框中心相对上次PnP移动超过 `8 px`；
- 靶框宽度相对上次PnP变化超过 `3%`。

PnP只处理四个角点，通常不是主要帧率瓶颈；YOLO、图像处理、激光检测和显示更可能占用主要时间。转弯时强制条件可能使PnP连续运行数帧，必须通过实机日志确认实际耗时。

## 8. STP23L读取、滤波和距离融合

### 8.1 STP23L滤波

当前 `Stp23lConfig`：

```text
max_jump_cm = 20.0
sample_window = 5
relock_frames = 3
relock_tolerance_cm = 3.0
stp_freshness_s = 0.30
stp_min_stable_frames = 3
stp_max_difference_cm = 15.0
```

处理逻辑：

- 最近样本使用中位数候选；
- 候选与当前稳定值相差不超过20 cm时正常更新；
- 大跳变不会永久丢弃，而是进入“新距离层级”候选；
- 新层级连续3帧彼此接近时允许重新接管；
- 这样可以从“先测到背景、后测到靶面”的状态恢复。

### 8.2 融合条件

满足以下全部条件时采用STP23L，否则采用PnP：

1. PnP距离有效；
2. STP23L数据在0.30 s新鲜时间窗内；
3. STP23L已经连续稳定至少3帧；
4. `abs(stp_distance - pnp_distance) <= 15 cm`；
5. 使用PnP距离预测出的激光点位于当前靶框内。

旧的固定 `stp_reference_x_px/y_px` 已删除。当前用预测激光点判断“测距方向是否落在靶面”。这个判断隐含一个重要假设：STP23L测距光轴与预测激光轴足够一致。如果STP23L传感器和照射激光并非同一轴线或安装位置明显不同，应恢复独立的测距轴标定，而不是继续使用预测激光点。

## 9. 激光预测坐标

当前横坐标固定：

```text
predict_x = 325.0 px
```

纵坐标使用用户实测的三次拟合，输入距离单位必须为cm，输出单位为px：

```text
predict_y =
    -7.04244406e-06 × dis_cm³
    +0.004392409042 × dis_cm²
    -0.908659639 × dis_cm
    +316.1899888
```

拟合优度：

```text
R² = 0.994112
```

示例：

```text
dis = 100.0 cm
predict = (325.0, 262.206)
```

预测结果会限制在图像范围内。

当前没有记录拟合采样距离的最小值、最大值和采样点分布。因此后续AI不能假定该三次式可以安全外推到任意距离。实机使用时应补充拟合有效距离范围，并对范围外距离选择限幅或回退策略。

## 10. 激光点检测和转弯重捕获

激光检测只在当前靶框的透视区域内执行：

- 快速透视图：`120×95`；
- 上一位置局部窗口：`40 px`；
- 快检连续失败3帧后允许一次完整兜底；
- 完整兜底尺寸：`240×190`；
- 每次完整兜底后冷却8帧；
- 靶框边缘6%区域清零，减少透视插值伪点；
- 当前使用One Euro滤波平滑激光坐标。

内部保存的不是最后原始像素，而是靶面归一化坐标：

```text
左上(0,0)  右上(1,0)
左下(0,1)  右下(1,1)
```

小车转弯或云台转动后，靶框在画面中会移动。上一激光点的原始像素不再可靠；归一化坐标可以根据当前靶框重新投影，作为局部重捕获中心。局部失败后仍必须扩大到整个靶面搜索。

紫外感光纸会保留旧光痕，红色中心线和环境高亮区域也可能干扰检测。实机应重点验证：

- 当前405 nm光斑与旧紫外痕迹能否稳定区分；
- 转弯运动模糊时是否误检；
- 激光靠近黑胶带边缘时是否被边缘屏蔽误伤；
- 连续照射形成大面积亮区时连通域筛选是否仍有效。

## 11. snapshot与丢标规则

`snapshot` 是一组加锁保护的最新控制数据：

```python
{
    "center_x": 0,
    "center_y": 0,
    "nowlaser_x": 0,
    "nowlaser_y": 0,
    "predict_x": 0,
    "predict_y": 0,
    "laser_dis": 0.0,
    "dis": 0.0,
}
```

作用：

- 视觉线程更新数据；
- UART线程读取一份完整副本；
- 避免UART读取到“部分字段属于旧帧、部分字段属于新帧”的混合状态；
- 当前帧缺少某字段时保持该字段最后有效值；
- 程序启动后尚无有效值时使用0或0.0。

具体规则：

- 新靶心有效时更新 `center_x/y`；
- 新激光点有效时更新 `nowlaser_x/y`；
- 新融合距离有效时更新 `dis` 和 `predict_x/y`；
- STP23L有新稳定值时更新内部 `laser_dis`；
- 丢标时不会把已有字段清零；
- 激光跟踪器内部的归一化位置与对外最后有效像素是两套不同状态。

当前UART帧没有有效位、帧序号和时间戳，因此M0无法判断一个坐标是本帧新值还是历史保持值。这是当前已接受但尚未消除的控制风险。

## 12. 数据输出接口

### 12.1 UART固定格式

用户已明确要求暂时保持：

```text
[cam,center_x,center_y,nowlaser_x,nowlaser_y,predict_x,predict_y,dis]
```

字段：

| 字段 | 单位 | 用途 |
| --- | --- | --- |
| `center_x/y` | px | 靶心 |
| `nowlaser_x/y` | px | 实际激光点 |
| `predict_x/y` | px | 基础题关闭激光时的预测瞄准点 |
| `dis` | cm | 当前融合距离，一位小数 |

UART按视觉帧号发送最新快照，不积压旧帧。M0已有对应解包代码，本阶段不改变帧结构。

### 12.2 终端调试格式

以 `0.20 s` 限频打印：

```text
[laser,x,y,],[center,x,y],[stp23l,xx.xcm],[dis,xx.xcm]
```

其中：

- `stp23l` 是原始稳定测距；
- `dis` 是融合后采用的距离。

### 12.3 当前严重已知问题：UART格式占位符数量错误

当前板端 `my_uart.py` 的格式字符串为：

```python
"[cam,%d,%d,%d,%d,%d,%d,%d,%.1f]"
```

它包含7个整数占位符和1个浮点占位符，但实际只传入6个整数坐标和1个距离，共7个参数。第一次执行发送时很可能抛出 `TypeError: not enough arguments for format string`，导致UART线程退出。

正确目标格式应只有6个 `%d`：

```python
"[cam,%d,%d,%d,%d,%d,%d,%.1f]"
```

这属于当前代码待修复项，不能因为 `py_compile` 通过就认为UART运行正常；格式化错误只会在实际执行 `serial_send()` 时暴露。

## 13. 基础题和发挥题控制逻辑

### 13.1 基础题

1. 激光保持关闭；
2. M0控制云台搜索；
3. RDK输出 `center`、`predict` 和 `dis`；
4. M0让 `center_x/y` 接近 `predict_x/y`；
5. M0结合视觉误差、电机反馈和稳定时间判断瞄准完成；
6. 稳定后才打开激光。

基础题不依赖 `nowlaser_x/y`，因为激光关闭时该字段只能保持历史值或初始0。

### 13.2 发挥部分（1）（2）

1. 激光连续发光；
2. RDK每帧输出靶心和实际激光点；
3. M0使用 `center-nowlaser` 作为视觉外环误差；
4. X42S内部位置闭环执行目标位置；
5. 转弯丢标时RDK使用归一化位置局部重捕获；
6. 局部失败后扩大到全靶搜索。

像素误差可近似换算为角度误差，但具体PID和角度增量限制属于M0任务，不在当前视觉代码中实现。

## 16. 主要难点和可能故障

### 16.1 几何和标定

- 27×18 cm必须对应同一条检测边界；
- 相机内参必须与运行裁剪和分辨率一致；
- 棋盘格标定畸变较大，边缘区域PnP可能更敏感；
- 平面PnP可能存在姿态分支和角点顺序问题；
- 靶框部分出画、胶带反光或透视过强时四角会退化；
- 激光和相机只近似平行，拟合可能随装配松动而失效。

### 16.2 距离融合

- STP23L可能测到背景、车体或靶外物体；
- “预测激光点在靶框内”只有在测距光轴与激光轴一致时才有效；
- STP与PnP都可能稳定但存在系统偏差，15 cm门限不能证明哪一个绝对正确；
- STP线程和视觉线程异步，0.30 s新鲜窗必须结合实际帧率验证；
- PnP强制事件在转弯时可能连续触发，需关注 `after` 阶段耗时。

### 16.3 激光检测

- 405 nm光斑在相机中的颜色可能随曝光、白平衡和感光纸变化；
- 紫外旧痕、红色圆线和镜面反光可能形成候选；
- 车体振动、转弯模糊和激光靠边会造成短时丢标；
- 保持最后有效坐标可能让M0把历史值误当实时值；
- One Euro滤波过强会增加控制延迟，过弱会造成坐标抖动。

### 16.4 控制和实时性

- 基础题2 s限制可能包含模型、相机或程序启动时间，现场计时口径尚未确认；
- UART线程当前存在格式占位符错误；
- UART没有换行、校验、有效位和时间戳，M0必须按既定流解析；
- 视觉、网页编码和UART同时运行时可能降低帧率；
- 电机动作延迟、相机帧延迟和M0外环叠加可能造成振荡；
- 发挥题连续光痕会增加视觉干扰，并可能影响最终评分痕迹。
