# Motor_CompareLastValue 初始零值比较

## 目标需求

将 `Motor_CompareLastValue()` 修改为初始历史值为 0、先比较再更新，使首次非零速度被识别为变化并返回 1。

## 涉及模块

- `code/Motor.c`
- `code/Motor.h`

## 实现思路

移除首次调用直接返回 0 的特殊分支。历史值数组保持静态零初始化；函数计算本次值是否不同后，更新对应通道历史值，并返回比较结果。

## 执行步骤

1. 更新函数实现，首次调用也按历史值 0 比较。
2. 更新函数声明注释，明确不同返回 1、相同返回 0 与先比较后更新的规则。
3. 对首次零值、首次非零、重复值和变化值进行验证。
4. 构建 MDK 工程；若本机缺少构建工具，记录原因并提供人工验证方法。

## 验证方法

- 首次输入 0 返回 0。
- 首次输入 20 返回 1。
- 连续输入 20 的第二次返回 0。
- 后续输入不同值返回 1。

## 验证结果

- 已按比较逻辑验证调用序列 `0, 20, 20, -20` 的返回值为 `0, 1, 0, 1`。
- 已使用 GCC 与 MDK 工程配置的全部包含目录对 `code/Motor.c` 进行语法检查，退出码为 0；主机 GCC 对 CMSIS 头文件报告的指针宽度警告与本次修改无关。
- 未执行 Keil MDK 完整构建：当前环境未安装 `UV4`、`UV5`、`armcc` 或 `armclang` 命令行工具。人工验证方法：在 MDK 中打开 `mdk/SeekFree_MSPM0G3519_Device_Library.uvprojx`，执行 Rebuild，并确认零错误。

## 已确认决定

- 采用方案 A：修改比较函数的公共语义，调用方保持不变。
- 只修改 `Motor.c` 与 `Motor.h`，不处理其他已发现的云台问题。
