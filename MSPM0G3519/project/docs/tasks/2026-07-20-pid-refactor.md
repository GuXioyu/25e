# PID 模块重构

## 目标需求

- 重构 `code/PID.c` 与 `code/PID.h` 的内部结构，保持位置式 PID 的全部现有控制能力。
- 不以现有外部调用接口为兼容约束；本次不修改外部调用模块。
- 使用 `mode_p`、`mode_i`、`mode_d` 控制比例、积分、微分项；模式值为 0 时禁用对应项。
- 积分限幅始终启用，并由每个 PID 实例的 `error_intmin` 与 `error_intmax` 配置边界。

## 涉及模块

- `code/PID.h`
- `code/PID.c`

## 实现思路

- 将比例、积分、微分的计算拆分为边界清晰的内部处理步骤。
- `mode_p` 仅作为 P 项启用标志；`mode_i` 在普通积分、变速非线性积分、积分分离中三选一；`mode_d` 在普通微分、微分先行、不完全微分中三选一。
- 删除文件级积分与微分临时变量，将运行时状态保存到 `PID_t`，避免多个 PID 实例共用状态而相互影响。
- 保留输入死区、输出偏置及输出限幅的控制能力，不新增 `PID_Reset()`。

## 执行步骤

1. 已定义 P、I、D 三项的互斥模式常量、模式字段与每实例微分输出状态。
2. 已重写 PID 更新流程，按启用模式计算 P、I、D 项，并在每次更新时限制积分值。
3. 已使用主机 GCC 编译并运行 PID 临时单元测试；外部调用调整不在本次范围内。

## 已确认决定

- 重构重点是结构清晰，不处理外部调用兼容。
- 所有现有控制能力保留。
- `mode_p` 仅启用或禁用 P 项，预留后续扩展空间。
- `mode_i` 的三种积分策略三选一。
- `mode_d` 的三种微分策略三选一，值为 0 时禁用 D 项。
- 不提供 `PID_Reset()`。
