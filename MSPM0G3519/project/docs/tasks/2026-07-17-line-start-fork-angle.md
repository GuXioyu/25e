# 启动直角弯完成角度

## 目标需求

小车启动后即进入的第一个左直角弯，使用独立的完成判定角度，避免启动加速度导致过早结束转弯。

## 涉及模块

- `code/Line.c`
- `code/Line.h`

## 实现思路

在 `Line.c` 顶部集中定义 `LINE_START_FORK_FINISH_ANGLE_DEG` 和 `LINE_FORK_FINISH_ANGLE_DEG` 两个宏，默认值分别为 `80.0f` 和 `65.0f`。`Line.c` 保留一次性启动直角弯标志：首次识别到左直角弯时保存 `80°` 为该次岔道的完成阈值并清除标志；后续左直角弯保存 `65°` 阈值。

## 执行步骤

1. 在 `Line.c` 集中定义启动首个直角弯与后续直角弯的可配置角度宏。
2. 为当前已锁定的直角弯保存完成判定阈值。
3. 首次识别直角弯时选择启动阈值，后续识别时选择常规阈值。
4. 更新 `Line.h` 中的逻辑说明。

## 验证方法

- 检查首次左直角弯锁定后使用 `LINE_START_FORK_FINISH_ANGLE_DEG`。
- 检查后续左直角弯锁定后使用 `LINE_FORK_FINISH_ANGLE_DEG`。
- 检查角度回绕处理保持不变。
- 工程未发现可运行的 `Line` 单元测试入口，完成代码静态检查后需上车验证：首次直角弯在偏航角差达到 `-80°` 后退出，后续直角弯在 `-65°` 后退出。

## 已确认决定

- 启动首个左直角弯完成判定角度：`80°`。
- 两个完成判定角度均以宏形式集中定义在 `Line.c`。
- 后续左直角弯默认使用 `65°` 完成判定角度。
