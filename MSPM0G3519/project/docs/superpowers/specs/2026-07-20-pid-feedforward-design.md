# PID 前馈项扩展设计

## 架构

前馈项是 `PID_t` 中与 P、I、D 项并列的独立控制项。它只依赖本次与上次误差，不新增历史状态，也不改变既有积分和微分策略。

## 模式与数据流

- `mode_f = PID_FMODE_DISABLE` 时，前馈输出为 0。
- `mode_f = PID_FMODE_ENABLE` 时，前馈输出为 `kf × (error_current - error_past)`。
- `PID_Update()` 先保存旧误差、计算当前误差，再依次计算 P、I、D、F 项。
- 最终输出为 P、I、D、F 项之和，之后沿用输出偏置、输出限幅和输入死区。

## 边界与错误处理

- `mode_f` 取值不是已定义的启用值时，前馈项不产生输出。
- `kf = 0.0f` 时前馈输出自然为 0。
- 前馈项不参与积分累积，也不改变 `dif_out`。

## 范围与验证

- 修改限于 `PID.h` 与 `PID.c`，不迁移外部调用。
- 使用临时主机测试先验证新接口在旧头文件中缺失，再验证启用、禁用与合成输出。
- 使用 GCC 严格警告选项独立检查 `PID.c`；MDK 构建不可用时记录限制。
