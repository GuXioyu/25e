# PID 模块重构设计

## 架构

PID 模块仅由 `PID.h` 与 `PID.c` 组成。`PID_t` 保存配置参数和单个控制器实例的运行时状态，`PID_Update()` 完成一次位置式 PID 更新。比例、积分和微分项在更新函数内按职责分段计算，最终统一处理输出偏置、输出限幅与输入死区。

## 模式与数据流

- `mode_p = 0` 时比例项为 0；非 0 时启用现有比例计算。
- `mode_i = 0` 时积分项为 0；其余值在普通积分、变速非线性积分、积分分离中三选一。每次积分计算后都以 `error_intmin`、`error_intmax` 强制限幅。
- `mode_d = 0` 时微分项为 0；其余值在普通微分、微分先行、不完全微分中三选一。
- 当前误差、历史误差、历史测量值和不完全微分的滤波结果均属于 PID 实例，禁止由文件级变量共享。

## 边界与错误处理

- 无效或未选择的模式不产生对应控制项。
- `ki = 0` 时清除积分累积；`kd = 0` 时微分项为 0。
- 输出继续按现有能力处理：计算 P、I、D 之和后施加输出偏置、输出上下限与输入死区。

## 范围

- 修改仅限 PID 模块；不修改云台或其他外部调用。
- 不提供 `PID_Reset()`。
- 不承诺与旧字段或旧调用方式兼容。

## 验证

- 静态核查三项模式的禁用、互斥选择与积分限幅路径。
- 确认两个 PID 实例的运行时状态不会互相覆盖。
- 运行工程可用的 MDK 构建；不可用时执行编译前检查并记录风险。
