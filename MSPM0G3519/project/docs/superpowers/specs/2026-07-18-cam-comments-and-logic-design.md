# Cam 模块注释补全与逻辑修复设计

## 范围

本次仅修改 `code/Cam.c` 和 `code/Cam.h`，并新增任务与设计文档。不会修改串口驱动或云台模块。

## 接口设计

保留 `CAM_TARGERT`、`CAM_LASER` 类型常量，坐标读取与新数据通知拆分为两个接口：

- `uint8_t Cam_GetFlag(void)`：读取新数据标志。标志为 1 时返回 1 并立即清零；否则返回 0。
- `uint16_t Cam_GetXY(uint8_t Axis, uint8_t Type)`：读取已经保存的单个坐标，不修改新数据标志。`Axis=1U` 表示 X，`Axis=2U` 表示 Y；参数无效返回 0。

## 数据流

1. UART4 接收完成标志置位后，`Cam_ReadXY()` 读取包缓存。
2. 模块验证标签为 `cam`，再解析两个十进制坐标字段。
3. 两个坐标均在 `uint16` 范围内时，提交目标坐标并置新数据标志。
4. 外部先调用 `Cam_GetFlag()` 消费本帧的新数据通知，再按需要调用 `Cam_GetXY()` 读取 X、Y 坐标。

不完整帧、错误标签、无效数值、超范围数值不会改变已保存坐标或新数据标志。

## 错误处理

- 未收到完整帧：`Cam_ReadXY()` 不处理。
- `Cam_GetFlag()` 在没有新数据时返回 `0U`。
- `Cam_GetXY()` 在轴或类型无效时返回 `0U`。
- 激光坐标仍为预留状态；读取激光类型不修改目标新数据标志。

## 验证

核对 Cam.c/Cam.h 的新接口声明和实现一致，并验证标志位只由 `Cam_GetFlag()` 清零。当前 Gimbal.c 未适配新的 `Cam_GetXY()` 签名，用户明确要求本次不修改它，因此完整工程构建会失败。
