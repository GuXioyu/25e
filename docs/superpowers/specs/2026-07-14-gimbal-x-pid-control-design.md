# 云台 X 轴 PID 与 Y 轴开环控制设计

## 目标

云台通过 UART4 接收 `[x,y]` 目标坐标。X 轴使用 `PID.c` 的位置式 PID 计算单帧角度增量，Y 轴继续使用像素误差到角度增量的固定比例控制。改动服务于 `REQ-SYS-02`、`TEST-B2/B3/F1/F2`，不扩展循迹、激光或视觉功能。

## 数据流

1. `Gimbal_ReadXY()` 检查 UART4 完整帧，在临界区内复制帧并清接收标志。
2. 接收层已经去掉方括号，函数直接从 `"x,y"` 解析两个无符号坐标。
3. 完整帧 `[0,0]` 表示未检测到目标，两轴保持当前位置并清除 X 轴 PID 状态。
4. X 轴 PID 目标直接使用 `ImgWidth / 2`，实际值为 x；PID 输出在使用处直接乘 `-1.0f` 后作为 yaw 相对角度命令。
5. Y 轴直接使用 `y - ImgHeight / 2` 计算偏差，按 `0.02 度/像素` 换算并限制为每帧 `±2 度`。

## 控制参数

- 图像尺寸：`640 x 360`，使用宽高除以 2 直接计算中心，不定义中心坐标宏。
- X 轴 PID：`Kp=0.02`、`Ki=0`、`Kd=0`，输出限幅 `±2 度`，死区 `2 像素`。
- Y 轴比例：`0.02 度/像素`，输出限幅 `±2 度`，死区 `2 像素`。
- 方向处理：不定义方向变量，直接使用相对角度命令的正负；X 轴在 PID 输出使用处乘 `-1.0f`。
- 两轴调用 `Motor_SetRelAngle()` 发送单帧相对角度增量；PID 和 UART 接口不变。

## 接口

- `Gimbal_Init(void)`：初始化两个电机和 X 轴 PID，不维护上电零位与累计目标角。
- `Gimbal_Task(void)`：每次消费一帧最新坐标，并更新需要动作的轴。
- `Gimbal_ReadXY(uint16_t *x, uint16_t *y)`：私有函数；有新帧返回 `1U`，无新帧或 UART4 无效返回 `0U`。
